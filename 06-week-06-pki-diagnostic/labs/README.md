# Week 6 Submissions — Quick Reference

This folder contains your completed lab write-ups and any artifacts generated during Week 6 labs.

---

## Folder Structure

```
labs/
  week-06/
    submissions/
      expired-certificate/
        lab-01-expired-certificate.md        ← Lab 01
      broken-chain/
        issuer_cert.pem                      ← Lab 02 (intermediate you retrieved)
        lab-02-broken-chain.md               ← Lab 02
      san-mismatch/
        lab-03-san-mismatch.md               ← Lab 03
      full-diagnostic-stretch/               ← Lab 04 (Stretch, optional)
        lab-04-full-diagnostic-stretch.md    ← Lab 04
```

---

## File Naming Convention

Lab write-ups must follow this format exactly:

```
lab-01-expired-certificate.md
lab-02-broken-chain.md
lab-03-san-mismatch.md
lab-04-full-diagnostic-stretch.md
```

Incorrect names will not be recognized by the lab tracker.

---

## Artifact Rules

- Commit any certificate files you retrieve or generate alongside your write-ups
- Use the `assets/` folder for screenshots only
- **Never commit private key files** — add `*.key` and `*_key.pem` to your `.gitignore`

---

## Submitting

1. Push all files to GitHub
2. Log in at [cvi-lab-tracker.lovable.app](https://cvi-lab-tracker.lovable.app)
3. Go to **Week 6** and submit each lab individually
4. Confirm the file path matches exactly before clicking Submit

> Pushing to GitHub alone does not mark the lab as submitted. You must click Submit in the tracker.
